# AGENTS.md

## Purpose
This repository is a Hugo static website with search powered by Pagefind.

In practice, it behaves as an npm wrapper around a Hugo build:
1. Build the site into `public/` using Hugo.
2. Run Pagefind against `public/` to generate search assets.
3. Serve with Hugo for local testing.

## What This App Is
- Site generator: Hugo (`hugo-bin-extended` in `package.json`).
- Search index + UI assets: Pagefind (`pagefind` npm dependency).
- Primary integration model: npm scripts orchestrate Hugo + Pagefind.

## Key Workflow
From `package.json`:
- `npm run build`
  - Runs `npm run hugo:build` (`hugo -d public`)
  - Then `npx pagefind --site public`
- `npm run serve`
  - Runs Hugo build
  - Runs `npx pagefind --site public --output-subdir ../static/_pagefind`
  - Starts `hugo server`

Why `serve` is special:
- It writes Pagefind assets into `static/_pagefind`, so Hugo serves them at `/_pagefind/*` during local development.

## Search Integration Points
- Pagefind UI partial: `layouts/partials/pagefind.html`
  - Loads `/_pagefind/pagefind-ui.css`
  - Loads `/_pagefind/pagefind-ui.js`
  - Initializes `new PagefindUI({ element: "#search" })`
- Header inclusion: `layouts/partials/page_header.html`
  - Includes the partial with `{{ partial "pagefind.html" . }}`

## Known Pitfalls And Recovery

### Pitfall: Pagefind asset paths drift from template paths
- Symptom: Search UI does not appear, or browser console reports missing `/_pagefind/pagefind-ui.js` or CSS.
- Cause: Output location for Pagefind assets changed without updating template references.
- Current expectation:
  - `layouts/partials/pagefind.html` references `/_pagefind/pagefind-ui.css` and `/_pagefind/pagefind-ui.js`.
  - `npm run serve` writes assets to `static/_pagefind` via `--output-subdir ../static/_pagefind`.
- Recovery:
  1. Confirm `npm run serve` still outputs to `static/_pagefind`.
  2. Confirm template paths still point to `/_pagefind/*`.
  3. Re-run `npm run serve` and hard-refresh the browser.

### Pitfall: Editing generated output in `public/`
- Symptom: Changes disappear after next build.
- Cause: `public/` is generated by Hugo and Pagefind.
- Recovery:
  1. Apply changes in source files (`content/`, `layouts/`, `static/`, config, scripts).
  2. Rebuild using `npm run build`.
